# L1 Requirements — Amalgam: Local Monolith Runner

## L1-1: Configuration System
The tool shall provide a YAML-based configuration system that defines all repositories, their types (microservice, shared library, UI plugin, dashboard), local paths, dependencies, and runtime settings so the user can declaratively describe their entire distributed system.

## L1-2: Local Package Resolution
The tool shall override NuGet (Nexus) package references for C# projects and npm package references for frontend projects, redirecting them to local source code paths so that all code runs from local checkpoints without publishing to any package registry.

## L1-3: Unified Backend Host
The tool shall compose all configured C# microservices into a single ASP.NET Core host process, merging their dependency injection registrations, middleware pipelines, and route mappings so the entire backend runs as one local monolith.

## L1-4: Frontend Plugin Linking
The tool shall link UI dashboard plugins from their local repos into the dashboard project using npm link (or equivalent workspace mechanism) so that frontend plugin development uses live local code.

## L1-5: CLI Interface
The tool shall provide a command-line interface with commands to initialize configuration, validate the setup, build all projects, run the unified host, and manage individual service toggles.

## L1-6: Developer Experience
The tool shall support selective service enabling/disabling, environment variable overrides, health-check output on startup, and clear error reporting so developers can iterate quickly on a subset of services.

## L1-8: Folder Merge / Assembly
The tool shall support merging multiple source folders from within a repository into a temporary staging directory before building or referencing the project, so that generated or split codebases can be used as unified libraries.

## L1-7: .NET Tool Packaging and NuGet Publishing
The CLI shall be packaged as a .NET global tool so users can install it via `dotnet tool install -g Amalgam.Cli` from nuget.org. The project shall include all required NuGet packaging metadata and support `dotnet pack` to produce a tool package.

## L1-10: Configuration Management API
The system shall provide an ASP.NET Core Web API backend that exposes RESTful endpoints for the configuration management web application. The API shall support CRUD operations on `amalgam.yml` configurations, validation, directory scanning, directory browsing for type-ahead, templates, cloning, and YAML preview/export. The API shall reuse the existing Core library services (ConfigLoader, ConfigValidator, DirectoryScanner, FolderMergeService).

## L1-9: Web Application for Configuration Management
The system shall provide an Angular web application using Angular Material that allows users to visually create, edit, validate, and manage Amalgam configuration files (`amalgam.yml`) through a responsive, mobile-first interface with designs for mobile (360px), tablet (768px), and desktop (1280px) breakpoints. The app shall use a dark mode theme by default. The app shall include comprehensive validation with descriptive error messages, type-ahead inputs, alerts, dialogs, notifications, configuration wizards, and support for templates and cloning configurations.
